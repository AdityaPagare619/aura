# Code of Conduct

## Our Pledge

AURA is a solo-developer project by Aditya Pagare. While public contributors are not currently solicited, this document establishes the professional standards and norms for any interaction related to this project — issue reports, pull requests, security disclosures, and community discussion.

We are committed to making participation in this project a respectful and harassment-free experience for everyone.

## Our Standards

**Behaviour that contributes to a positive environment:**

- Using welcoming and inclusive language
- Being respectful of differing technical viewpoints and experience levels
- Gracefully accepting constructive criticism
- Focusing on what is best for the project's mission: private, on-device AI
- Showing empathy towards other contributors and users

**Behaviour that is not acceptable:**

- Harassment of any kind, public or private
- Trolling, insulting comments, or personal attacks
- Publishing private information (doxxing) of any person without explicit permission
- Deliberate attempts to undermine the project's privacy-first mission (e.g., advocating for cloud integrations or telemetry)
- Submitting issues or PRs in bad faith (spam, intentional breakage, supply-chain attacks)

## Scope

This Code of Conduct applies within all project spaces — GitHub Issues, Pull Requests, Discussions, the Security disclosure channel, and any other space where the project is represented.

## Reporting

If you observe or experience unacceptable behaviour, please contact the project maintainer directly:

**Email:** See [SECURITY.md](SECURITY.md) for contact details.

All reports are reviewed promptly. The maintainer will respond to good-faith reports within 7 days.

## Enforcement

The maintainer reserves the right to:

- Remove comments, commits, issues, or other contributions that violate these standards
- Temporarily or permanently block any contributor for sustained or severe violations
- Label issues as `invalid` or `wontfix` for bad-faith submissions
