- With ef_construction=200 and 10K nodes this is extremely expensive — suitable only for offline/thermal-aware consolidation windows (comment says so), but no guard prevents in-path calls
- Fix: Add a #[cold] annotation + assertion that it's only called from a consolidation task
H4 — INFO: to_bytes() pre-allocates nothing, grows Vec dynamically
- buf.extend_from_slice() in a tight loop with no with_capacity() — repeated reallocation during serialization of large indexes
- Fix: Pre-calculate buf.reserve(estimated_size) from header constants
H5 — INFO: LCG seed is hardcoded 42 — deterministic across restarts
- All HNSW instances start with rng_state = 42; level generation is therefore identical for the same insertion sequence
- This is benign for correctness but means all databases have the same graph topology after compaction, which could lead to pathological clustering for certain embedding distributions
- No action needed, but worth noting
