5. Token Economics Analysis
Parameter	Value	Assessment
Model context window	32,768 tokens	Generous
DEFAULT_CONTEXT_BUDGET	2,048 tokens	Severely conservative — 6.25% utilization
RESPONSE_RESERVE	512 tokens	Reasonable for action JSON

Assessment: The 2048-token budget is the single biggest operational constraint. For a ReAct agent that needs to reason about screen content, conversation history, user goals, and tool schemas, 1536 usable tokens forces aggressive truncation on most real interactions. The progressive truncation in context.rs:466-476 is well-designed (correct priority ordering), but it will activate on nearly every multi-turn conversation. Increasing to 4096-8192 would dramatically improve reasoning quality with minimal latency impact on modern devices running Q4 quantized models.
The dual tracking systems (daemon TokenBudgetManager vs neocortex TokenTracker) create a risk of budget disagreement — the daemon might think it has budget remaining while neocortex is already truncating, or vice versa.

6. Top 5 Recommendations
1. Enable GBNF at decode time, not just post-hoc validation
Impact: Critical | Effort: Medium
The grammar infrastructure exists but is only applied as a post-generation validator. llama.cpp's llama_sampler_init_grammar() should be used to constrain token generation to valid JSON/action schemas. This eliminates malformed outputs rather than penalizing them after wasting compute.
2. Increase DEFAULT_CONTEXT_BUDGET to 4096-8192
Impact: High | Effort: Low
The current 2048 budget wastes 94% of the model's context window and forces truncation on nearly every real interaction. A Q4_K_M 8B model at 4096 tokens adds negligible latency. This single change would most improve real-world output quality.
3. Fix the const-to-mut FFI cast
Impact: Critical | Effort: Low
Replace tokens.as_ptr() as *mut LlamaToken with a mutable buffer copy, or verify llama.cpp's llama_batch_get_one contract guarantees read-only access and document it. As-is, this is technically undefined behavior.
4. Unify token budget tracking
Impact: High | Effort: Medium
Either have the daemon delegate all budget management to neocortex via IPC, or synchronize the two trackers. The current dual-system with different heuristics will produce drift and unpredictable truncation behavior.
5. Activate or remove the RouteClassifier
Impact: Medium | Effort: Low-Medium
The 10-node classifier cascade in classifier.rs is well-implemented dead code. Either wire classify_task() to actually use it (enabling DGS for simple tasks, saving LLM calls), or delete the 441 lines to reduce maintenance burden. The current always-return-SemanticReact approach sends every task through the full LLM pipeline, which is wasteful for simple commands.
