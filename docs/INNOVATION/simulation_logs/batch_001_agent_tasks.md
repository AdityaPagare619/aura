## Simulation Process for Each Agent
For each assigned persona, agents will:
1. Review the complete persona profile
2. Assign a random context/topic for thought generation
3. Generate an authentic thought based on persona filters
4. Create platform-specific expression variant
5. Generate emotional reaction and potential actions
6. Save complete simulation record to logs
7. Report completion status

## Expected Outputs per Agent
- 5 simulation records (one per persona)
- Each record includes: timestamp, persona ID, full profile, context, thought, expression variant, metadata
- Saved to individual log files in the batch directory
- Completion status reported to batch coordinator

## Quality Checks
- Ensure persona diversity is maintained
- Prevent stereotyping through individual variation
- Balance positive, negative, and neutral perspectives
- Verify platform-appropriate expression styles
- Confirm authentic voice matching persona background
